HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Used to create the structure of the webpage.
  • It is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • HTML elements are usually composed of an opening tag and a closing tag represented with angle brackets.
  • 3 Common Elements types : Meta-related elements, Content division elements, and Semantic elements.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Use the "*" selector when you want to apply a rule to all the elements visible on the page.
  • You can combine two or more selectors using a "," to avoid Do Not Repeat Yourself (DRY) code.
  • Besides text, we can also apply CSS rules to images!
  • Remember to link your css style sheet before the title element.

Git

  • A version control system that lets you manage and keep track of each version of your project.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git status: checks what branch we are currently on.
  • git add [file]: add a file as it looks now to your next commit.
  • git commit -M [descriptive message]: commit your staged content as a new commit snapshot.
  • git pull: command to recieve a branch's modifications into the local environment.

JavaScript

  • A fuction is simply a set of instructions that tells the computer how to perform a certain task.
  • To call a function, we use the name of the function in our JavaScript file, and list at the bottom of the .js file.
  • Always double check and make sure the fuction matches the array.
  • Add messages to better explain the output in the console by placing before calling the fuction at the bottom of .js.
  • To pick randomly from your array you can use Math.random().
